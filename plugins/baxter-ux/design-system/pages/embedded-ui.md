## Embedded UI



#### Design Principles

Embedded components are designed to support:

Clarity: Simple, legible interfaces optimized for quick recognition

Responsiveness: Real-time feedback and minimal latency

Efficiency: Compact layouts suited for limited screen space

Consistency: Alignment with foundational styles and cross-platform UX patterns







### Component Structure

Embedded UI components are organized into two categories:



#### Base Components

Essential building blocks for device interfaces:

Avatar

Buttons

Controls

Input

List

Search Bar

...



#### UX Pattern Components

Predefined flows and layouts for common user tasks:

Alarms

Filtering and sorting

Navigation bars and side panels

Search and results

Dialogs & Overlays

...



Each component is built on the Foundation Library, ensuring visual consistency and functional reliability across embedded platforms.







### Using the Figma UI Kit

To begin designing for Embedded UI, product teams should add the following libraries to their Figma file:

Foundation Library
Includes shared design tokens and styles (color, typography, spacing, etc.)

Embedded UI Kit
Contains all base and pattern components specific to embedded systems

This setup enables access to both the foundational style guide and the full set of embedded-specific components, supporting efficient and consistent design workflows across device interfaces.
