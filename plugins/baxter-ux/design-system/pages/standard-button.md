### Style Variants

The button component has four main style variants, that are then expressed with color variations. This differentiation allows designers to create a visual hierarchy when placing available actions on a screen, as documented in Usage Guidelines.



#### Regular buttons

These variants are designed for general use across the interface.





#### Neutral

These variants are intended for specific contexts, such as colored backgrounds or alert messaging components.



#### Destructive

These variants are intended for actions with irreversible outcomes.







### Interaction States

State variants reflect the button’s interaction status and provide visual feedback to users during their journey.
