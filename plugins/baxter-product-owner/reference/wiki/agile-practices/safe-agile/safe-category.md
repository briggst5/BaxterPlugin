# SAFe Category
Under each work item type in DevOps, there is a field called "SAFe Category," with a list of the following four items:
- Capability
- Enabler
- Feature
- Solution


## Enabler:
- An enabler is a backlog item that helps extend the architectural runway or improve the performance of the development value stream.

- Enablers can include technical work, exploration, or compliance activities.

## Capability
- A capability is a high-level solution that typically spans multiple Agile Release Trains (ARTs).
- It describes what a system can do and is sized to fit within a Planning Interval.
- Capabilities include a benefit hypothesis and acceptance criteria.

## Feature
- A feature is a service that fulfills a stakeholder need.
- There are two types: **Business Features** (customer-centric) and **Enabler Features** (support future requirements).
- Each feature includes a benefit hypothesis and acceptance criteria and is sized for delivery by a single ART in one Planning Interval.

## Solution
- A solution is the complete set of features and capabilities that deliver value to the customer.
- It is built by multiple ARTs and may include various components and systems working together.
